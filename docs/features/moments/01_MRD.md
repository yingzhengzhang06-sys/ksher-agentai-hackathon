# 发朋友圈数字员工 MRD

> 文档版本：v0.1  
> 功能编号：MRD-001  
> 功能名称：发朋友圈数字员工  
> 内部模块名：朋友圈转发助手  
> 所属项目：市场专员数字员工 / 渠道商内容工厂  
> 当前阶段：MVP 需求定义  
> 使用对象：产品负责人 / 产品经理 / UI/UX / AI 能力设计师 / 架构师 / Codex / Claude Code  

---

## 1. 背景

当前项目基于已有黑客松项目继续深化，不是从零新建项目。

第一阶段目标不是构建完整市场工作台，也不是做全能型数字员工，而是先围绕一个高频、清晰、可验证的工作动作，跑通完整产品研发闭环：

想法 → MRD → PRD → UI/UX → AI 能力设计 → 技术方案 → 开发任务 → 开发 → 测试 → 上线 → 复盘

本阶段选择的样板功能是：发朋友圈数字员工。

它的核心任务是帮助跨境支付渠道商、区域 BD、市场专员等用户，在较短时间内生成一条专业、克制、合规、可复制到朋友圈的内容草稿。

## 2. 一句话定位

帮助跨境支付渠道商在 3 分钟内生成一条可以复制到朋友圈的专业、合规、面向商户的营销内容草稿。

## 3. 目标用户

### 3.1 第一优先级用户

#### 用户 A：城市代理商 / 区域渠道商

特征：

- 有本地商户资源
- 懂销售，但不擅长内容生产
- 日常通过微信、朋友圈、社群触达客户
- 需要持续发布专业内容来建立信任感

核心诉求：

> 每天有内容可发，最好可以直接复制。

#### 用户 B：跨境服务商渠道伙伴

包括：

- ERP 服务商
- 物流服务商
- 财税服务商
- 培训机构
- 独立站服务商

特征：

- 已经服务跨境商户
- 希望把支付服务作为附加价值
- 不希望投入太多内容生产成本
- 不熟悉金融和跨境支付表达边界

核心诉求：

> 我想顺手推广支付产品，但不要让我自己研究金融话术。

### 3.2 第二优先级用户

#### 用户 C：渠道负责人 / 市场专员 / 销售支持人员

特征：

- 需要批量支持多个渠道商
- 经常被问要素材、要话术、要案例
- 希望把重复性的内容支持自动化

核心诉求：

> 让渠道商自助拿内容，减少总部反复支持压力。

---

## 4. 用户痛点

### 4.1 渠道商不会持续产出内容

很多渠道商有客户资源，但没有持续内容运营能力。

常见做法包括：

- 转发总部公众号
- 复制竞品文案
- 临时让销售自己写
- 用通用 AI 生成泛泛内容

这些方式的问题是：

- 不够贴近渠道商自己的客户
- 不够专业
- 不够稳定
- 不像真实朋友圈表达
- 没有形成长期内容节奏

### 4.2 跨境支付内容有专业门槛

跨境支付涉及：

- 多币种
- 汇率
- 到账时效
- 费率
- 清算路径
- 支付牌照
- 合规要求
- 竞品对比
- 电商平台规则

普通渠道商很难把这些内容讲清楚。

结果是：

> 产品有价值，但渠道商讲不明白。

### 4.3 金融和跨境支付内容存在合规风险

跨境支付内容不能随便写。

常见风险包括：

- 使用“最低费率”“最安全”“最快到账”等绝对化表达
- 暗示保证收益
- 夸大到账时效
- 夸大资金安全能力
- 暗示规避外汇监管
- 使用未经授权的平台 Logo 或竞品名称
- 对客户案例数据进行夸大表达

因此，内容生成不能只追求“好看”，还必须保证：

> 专业、克制、可发布、低风险。

### 4.4 总部素材供给跟不上渠道商节奏

传统内容支持流程通常是：

渠道商需要内容 → 找总部要素材 → 总部排期 → 等待设计 / 文案 / 审核 → 错过发布时机

本功能希望把流程压缩为：

渠道商输入场景 → AI 生成可转发文案 → 基础合规检查 → 人工确认后复制使用

## 5. 使用场景

### 场景 1：渠道商日常朋友圈更新

用户想保持朋友圈活跃，但不知道今天发什么。

典型需求：

> 帮我生成一条适合发给跨境电商卖家的朋友圈内容。

### 场景 2：围绕某个产品卖点做轻量推广

用户想表达某个产品价值，例如：

- 到账快
- 费率透明
- 合规安全
- 多平台收款
- 东南亚本地币种支持

典型需求：

> 我想发一条关于“到账效率”的朋友圈，但不要太像广告。

### 场景 3：面向不同商户类型定制表达

不同目标客户痛点不同：

| 目标客户 | 内容重点 |
|---|---|
| Amazon 卖家 | 多站点收款、提现、汇率、资金周转 |
| Shopee 卖家 | 东南亚、本地币种、平台收款、到账速度 |
| 外贸 B2B | 电汇成本、账期、合规收款、大额付款 |

系统需要根据目标客户差异生成不同内容，而不是所有选项输出都差不多。

### 场景 4：市场专员批量支持渠道商

市场专员需要给多个渠道商提供可转发内容。

目标不是替代市场专员，而是减少重复性文案支持工作。

---

## 6. 当前替代方案

### 6.1 转发总部内容

优点：

- 相对安全
- 品牌统一

缺点：

- 不够贴近具体渠道商
- 不够私域化
- 更新频率有限
- 用户容易觉得像官方广告

### 6.2 渠道商自己写

优点：

- 更像本人表达
- 更贴近客户

缺点：

- 专业性不足
- 合规风险高
- 产出不稳定
- 时间成本高

### 6.3 使用通用 AI 写文案

优点：

- 生成速度快
- 使用门槛低

缺点：

- 不懂跨境支付行业
- 不懂渠道商场景
- 容易生成空泛内容
- 容易出现夸张和不合规表达
- 缺少产品知识和风格约束

---

## 7. 产品机会

朋友圈是渠道商最常用的轻量触达方式。

相比短视频、直播、活动策划，朋友圈具备：

- 低门槛
- 高频
- 成本低
- 即写即用
- 适合私域信任建立

因此，发朋友圈数字员工适合作为市场专员数字员工的第一个 MVP 功能。

它不是普通 AI 写作工具，而是：

> 面向跨境支付渠道商的专业营销内容生成工具。


---

## 8. MVP 范围

第一阶段只做：

### 8.1 朋友圈文案生成

用户输入基本场景后，系统生成一条适合复制到朋友圈的内容草稿。

### 8.2 渠道商转发建议

系统给出轻量转发建议，例如：

- 适合发给谁看
- 适合什么语气
- 是否建议加一句私信 CTA
- 是否建议人工再确认某个事实

### 8.3 基础合规检查

系统对输出内容做基础风险提示。

风险类型包括：

| 风险类型 | 示例 |
|---|---|
| 绝对化表达 | 最低、最快、最安全、100% |
| 收益承诺 | 保证收益、稳赚、无风险 |
| 夸大安全 | 绝对安全、永不冻结 |
| 违规外汇暗示 | 规避监管、突破额度 |
| 缺少条件说明 | T+0 到账但未说明适用条件 |

合规结果分为：

- 可发布
- 建议修改
- 禁止发布

### 8.4 复制与重新生成

用户可以：

- 复制生成文案
- 重新生成另一个版本
- 基于输出进行人工修改

### 8.5 有用 / 没用反馈

用户可以对生成结果进行简单反馈：

- 有用
- 没用
- 可选填写原因

该反馈用于后续优化 Prompt、风格和内容质量。

---

## 9. MVP 输入字段

第一版输入字段如下：

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---:|---|
| 内容类型 | 下拉 | 是 | 产品解读 / 热点借势 / 客户案例 |
| 目标客户 | 下拉 | 是 | Amazon 卖家 / Shopee 卖家 / 外贸 B2B |
| 产品卖点 | 多选 | 是 | 到账快 / 费率透明 / 合规安全 |
| 文案风格 | 单选 | 是 | 专业 / 轻松 / 销售感强 |
| 补充说明 | 文本框 | 否 | 用户可输入额外背景 |

字段设计原则：

- 不让用户填写过多信息
- 控制输入复杂度
- 优先用选项降低使用门槛
- 保留一个补充说明字段用于表达个性化需求

---

## 10. MVP 输出内容

第一版输出包括：

| 输出项 | 说明 |
|---|---|
| 标题 / 首句 | 适合作为朋友圈第一句话 |
| 正文 | 可直接复制的朋友圈文案 |
| 转发建议 | 告诉渠道商适合发给谁、怎么发 |
| 合规提示 | 判断是否可发布，以及是否有风险表述 |
| 改写建议 | 如果存在风险，给出更安全表达 |

## 11. MVP 操作按钮

第一版操作包括：

| 操作 | 说明 |
|---|---|
| 生成内容 | 生成朋友圈文案 |
| 复制文案 | 复制正文内容 |
| 重新生成 | 生成另一个版本 |
| 有用 | 记录正反馈 |
| 没用 | 记录负反馈 |

## 12. 本期不做范围

第一阶段明确不做以下内容：

### 12.1 不做自动发布

原因：

- 涉及微信生态限制
- 涉及账号权限
- 容易引入复杂授权问题
- 当前验证重点不是自动发布，而是内容是否可用

### 12.2 不做定时发布

原因：

- 需要任务调度、账号授权、失败重试等能力
- 会显著提高工程复杂度
- 当前 MVP 不需要验证发布自动化

### 12.3 不做审批流

原因：

- 第一阶段是单用户内容生成验证
- 不涉及团队协作和多级审核
- 审批流会拉高产品复杂度

### 12.4 不做效果追踪闭环

原因：

- 需要真实发布数据或用户回填
- 第一阶段无法稳定获得真实转发和互动数据
- 当前先验证内容生成质量

### 12.5 不做图片海报生成

原因：

- 海报设计涉及视觉质量和品牌规范
- 当前阶段文案价值更容易验证
- 图片生成可作为后续独立能力

### 12.6 不做短视频脚本

原因：

- 短视频涉及平台、分镜、拍摄、剪辑、发布策略
- 复杂度远高于朋友圈文案
- 适合在朋友圈能力稳定后再做

### 12.7 不做完整渠道商门户

原因：

- 用户权限、素材管理、数据看板都会提高复杂度
- 第一阶段只验证单点价值

## 13. 内容质量标准

一条合格的朋友圈文案应满足以下标准。

### 13.1 可读性

- 3 秒内知道在讲什么
- 不堆砌专业词
- 不像公文
- 不超过 300 字

### 13.2 专业性

- 能体现跨境支付行业理解
- 能讲出具体场景
- 能解释一个实际痛点
- 不只是泛泛营销话术

### 13.3 私域表达

内容不能像公司公告，也不能像品牌宣传稿。

它应该具备：

- 朋友圈口吻
- 私域表达
- 有人味
- 不油腻
- 不夸张
- 不违规

错误示例：

> Ksher 是全球领先的跨境支付平台，为您提供最安全、最低费率、最快到账的收款方案。

问题：

- “全球领先”缺乏依据
- “最安全”“最低费率”“最快到账”属于高风险绝对化表达
- 像广告，不像朋友圈

更合适的表达：

> 最近不少做东南亚电商的朋友都在问：收款速度和换汇成本到底怎么平衡？其实选收款工具时，不能只看表面费率，还要看到账时效、汇率透明度和合规路径。需要的话，我可以帮你做一份简单对比。

### 13.4 转化性

至少包含一个轻量 CTA，例如：

- 需要的话，我可以帮你做个对比
- 想了解东南亚收款方案，可以私信我
- 如果你也遇到类似问题，可以聊聊

### 13.5 合规性

- 不使用绝对化表达
- 不承诺收益
- 不夸大安全
- 不误导用户规避监管
- 不伪造客户案例
- 不虚构未经确认的数据

## 14. 成功指标

### 14.1 第一阶段指标

| 指标 | 目标 |
|---|---|
| 生成次数 | 种子用户每周至少生成 3 次 |
| 复制率 | 生成后复制比例 ≥ 50% |
| 有用率 | 用户点击“有用”比例 ≥ 60% |
| 负反馈原因收集率 | “没用”后至少能收集原因 |

### 14.2 第二阶段观察指标

以下指标只观察，不作为第一阶段上线门槛：

| 指标 | 说明 |
|---|---|
| 实际转发率 | 需要用户主动反馈或后续接入数据 |
| 人工修改率 | 用于判断生成内容是否越来越接近可用 |
| 复用率 | 判断同一用户是否持续使用 |
| 渠道商满意度 | 通过访谈判断是否节省时间 |

## 15. 验证计划

### 15.1 种子用户

建议选择 3-5 个用户测试：

| 用户类型 | 数量 |
|---|---:|
| 新代理商 | 1-2 |
| 老代理商 | 1-2 |
| 渠道负责人 / 市场专员 | 1 |

### 15.2 测试任务

每个用户完成以下任务：

1. 选择目标客户
2. 选择产品卖点
3. 生成朋友圈内容
4. 判断是否愿意直接发
5. 说明不满意的地方
6. 复制或重新生成一次

### 15.3 访谈问题

测试后访谈：

1. 这条内容你愿意发朋友圈吗？
2. 哪一部分让你觉得不像你自己会说的话？
3. 内容是否太广告？
4. 有没有专业但看不懂的词？
5. 有没有你担心不合规的表达？
6. 如果每天给你 1 条这样的内容，你会不会用？
7. 你希望它下一步增加什么能力？

## 16. 风险与假设

### 16.1 核心假设

| 假设 | 验证方式 |
|---|---|
| 渠道商有持续发朋友圈需求 | 种子用户访谈 |
| 渠道商缺少专业内容能力 | 观察其现有朋友圈 |
| AI 生成内容可降低总部支持压力 | 对比生成前后沟通频次 |
| 合规检查是差异化价值 | 观察用户是否重视合规提示 |

### 16.2 主要风险

| 风险 | 说明 | 应对 |
|---|---|---|
| 文案太像 AI | 用户不愿意发 | 增加去 AI 味和风格控制 |
| 内容太广告 | 朋友圈用户反感 | 强化私域表达模板 |
| 合规误判 | 影响发布效率 | 允许人工覆盖修改 |
| 输入太复杂 | 用户不愿意填 | 控制字段数量 |
| 输出太泛 | 缺少业务差异 | 增强场景知识库 |
| 范围膨胀 | 变成完整市场工作台 | 坚持第一阶段只做草稿生成 |

## 19. 当前版本结论

发朋友圈数字员工适合作为市场专员数字员工的第一个 MVP 功能。

原因：

1. 高频
2. 易验证
3. 与渠道商赋能强相关
4. 可快速开发
5. 后续可扩展到素材、视频、活动和数据体系

当前阶段不应做大而全的市场专员工作台，而应先把：

> 生成一条渠道商愿意复制并人工确认后发布的朋友圈内容草稿

这个动作打磨到足够好。
