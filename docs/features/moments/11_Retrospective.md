## 3. 关键问题与处理

### 3.1 MOMENTS-QA-01

- 问题：M-07 AI 异常兜底路径被同 session 限频拦截。
- 根因：异常兜底验证与同 session 限频用例互相污染。
- 处理：
  - 前端使用浏览器级 `moments_session_id`。
  - 新增独立 session 下 `mock:error` 应返回兜底而非限频的回归测试。
  - QA 使用独立浏览器会话复验 M-07。
- 结果：已关闭。

### 3.2 QA-BLOCK-02

- 问题：Codex 无法替代真实移动端人工验收。
- 影响：M-08 剪贴板权限、真实触控和 WebView 行为仍需人工确认。
- 当前处理：Ian 已使用手机端访问局域网页面并确认基本合格，生成、复制、重新生成等核心交互无阻塞问题。

## 5. 暴露的问题

| 问题 | 影响 | 后续建议 |
|---|---|---|
| Streamlit 组件在 Playwright 下不总是稳定可控 | 多选、剪贴板、真实触控仍需人工验证 | 保留人工移动端验收门禁；后续考虑更稳定的浏览器自动化策略 |
| Chrome headless 可能只捕获骨架屏 | 不能替代内容级验收 | 将 headless 截图定位为服务可访问检查，不作为 QA 通过依据 |
| 文档数量快速增加 | 对一人公司可能造成维护负担 | 用 `08_Collaboration_Status.md` 做当前事实源入口，其他文档归档引用 |
| 历史未提交文件较多 | 容易混入无关提交 | 每次提交必须精准 `git add <files>`，禁止 `git add .` |
| 真实 AI 默认链路仍需上线治理 | 当前本地真实 AI smoke test 通过，但正式上线还需要质量、成本和限流策略 | 上线前补充质量抽样、成本监控和限流策略 |

## 6. 协作经验

- 产品负责人适合控制范围和门禁，不应直接推动无边界开发。
- 架构师应在缺陷修复前先裁决“改哪里、不改哪里”。
- 工程师只在任务明确、文件范围明确、测试命令明确时动手。
- QA 不应让自动化截图替代人工验收记录。
- 项目助理 Bot 适合维护当前状态、下一角色和阻塞项，但不做产品或技术裁决。
