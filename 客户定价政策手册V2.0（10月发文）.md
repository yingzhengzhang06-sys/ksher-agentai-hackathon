# 前言

为确保开时支付提供的产品和服务定价与价值相符，在市场具备竞争力、激励不同层级的商户通过开时支付实现更多的交易量，同时为各条线提供标准、清晰的定价参考和特殊价格审批流程参考，特发布本规则。

## 一、范围与定义

1. 规则适用范围
    
    1. 包含“对客户品标准定价”和“对客户品分层定价”两套原则：
        
        - 标准定价：通过网站、信息页等公开资料发布的直客/渠道商户定价（含手续费和汇率）。
            
        - 分层定价：内部参考的优惠定价，根据实际/潜在交易量分级提供，不公开发布。
            
2. 客户分级标准

核心原则： 以DEF等级客户为基准价，每升高一个等级，汇率优化。

|---|---|---|---|---|---|
|客户等级|印尼/马来|菲律宾|泰国|越南|香港全球收（B2B）|
|优化幅度|   |   |   |   |
|S|0.40%|0.07%|0.09%|0.70%|0.07%|
|A|0.30%|0.06%|0.06%|0.60%|0.06%|

公式： `报价汇率 = DEF基准汇率 * (1 + 等级优化幅度)`

2. 报价规则
    

- 业务场景：B2C和B2B，不包括B2S。
    
- 源币种与目标币种： 支持各国家站点本地币种兑换为USD或CNY。
    
- 前提条件： 必须先完成费率优化，若客户仍因汇率原因无法合作，方可申请汇率优化，销售需在申请时说明已进行的费率优化措施。
    
- 特批客户标准建议：
    
    - 建议标准： 单月潜在交易量持续 ≥ 500万美金 的客户，可申请特批汇率。
        
    - 审批流程： 走“非标申请流程”，需Donald审批。
        

## 九、价格审批流程

### （一）费率审批流程

1. 执行细则
    
    1. 交易量认定
        
        1. 以客户声明可迁移至Ksher的月度上账金额（美金）为准
            
        2. 虚报交易量将列入黑名单
            
    2. 季度Review
        
        1. 未达标的客户经沟通后按实际交易量调整费率
            
2. 标准定价流程
    
    1. 流程：销售提交申请 → 商服直接配置
        
    2. 适用条件：费率在公司标准定价区间内，无需特殊审批
        
3. 特殊情形审批
    
    1. 费率＞0.1% 销售申请 → 区域经理审批 → 商服配置
        
    2. 费率≤0.1% 销售申请 → 区域经理审核 → 销售VP批准 → 商服配置
        
4. 竞对响应规则
    
    1. 适用条件
        
        1. 仅限D级及以上客户
            
        2. 需提供书面竞对报价证明
            
    2. 价格匹配原则
        
        1. 优先上调1-2个等级档位的价格，如C级商户在有竞对的情况下，可优先申请B级价格，其次是A级
            
        2. 超过2个等级档位的价格审批，需要到VP层级加批
            
    3. 禁止条款
        
        1. E级客户不开放价格竞争
            
        2. 不得跨业务线比价（B2C/B2B独立核算）
            

### （二）汇率申请流程

1. 汇率报价的系统支持：
    
    1. 报价从人工转自动化：上线不同客户等级汇率报价的bi看板，以bi看板报价为准；
        
    2. 销售查询支持：crm上线客户的汇率报价的字段信息，对马来印尼未进行普调商户上线标签字段支持`印马存量商户未普调` ；
        
2. 标准分层汇率申请流程（适用于C/B/A等级）：
    
    1. 销售发起： 销售在飞书审批中提交申请，并附上已进行费率优化的说明。
        
    2. BDM审批： 直属BDM进行一级审批，评估客户潜力和申请合理性。
        
    3. 区域经理审批： 区域经理进行二级审批，把控区域整体价格水平。
        
    4. FX配置： 审批通过后，流转FX团队进行系统配置。
        
3. 非标汇率申请流程（适用于特殊情况）：
    
    1. 销售发起： 销售在系统/工单中提交申请，必须明确说明客户体量及特批理由。
        
    2. BDM审批： 直属BDM进行一级审批。
        
    3. 区域经理审批： 区域经理进行二级审批。
        
    4. Donald审批： 由Donald进行最终裁定。
        
    5. FX配置： 审批通过后，流转FX团队进行系统配置。
        
4. 审批结果： 若任何一环审批人拒绝，申请将被退回给销售，并说明理由。
    

### （三）Review与考核机制

1. 客户定级Review机制
    
    1. Review周期： 每3个自然月进行一次。
        
    2. 数据依据： 使用调整后第三个月的实际交易量进行考核。
        
        1. 示例： 客户10月申请了C级优惠，则考核其12月份的实际交易量是否达到5万美金（C级最低标准）。
            
    3. 执行动作：
        
        1. 达标/超标： 维持原等级或根据规则探讨是否升级。
            
        2. 未达标： 系统自动或在运营人员操作下，于次月将其汇率和费率回调至其实际交易量对应的等级。销售需提前通知客户。
            
2. FTB数据监控与考核
    
    1. 监控范围： 按区域、团队、个人维度监控FTB（遵循基准）指标。
        
    2. 复盘频率： 月度复盘，在销售管理会议上公布各团队/个人的FTB达标率。
        
    3. 考核联动： FTB指标将纳入销售的分润/绩效奖金考核体系，直接与收入挂钩，以驱动销售遵守报价规则。
        

### （四）生效、修订与解释

1. 生效时间：本定价规则自其正式发布之日起生效执行。
    
2. 规则修订：本公司将依据市场动态、业务发展需要及定期的运营数据复盘，对本规则进行不定期的调整与更新。
    
3. 效力优先：规则的历次修订版本均以最新发布的文件内容为准，此前所有与之冲突的旧版规则或文件自动失效。
