# 24_AI_Quality_Sampling_Runbook - 真实 AI 输出质量抽样手册

> 文档版本：v0.1
> 生成日期：2026-04-25
> 生成角色：QA 测试工程师 / 产品负责人 / AI 工程师
> 关联任务：`AI-GOV-03`
> 当前状态：抽样手册，不修改业务代码，不读取或输出密钥

---

## 1. 目标

本 Runbook 用于对“发朋友圈数字员工”的真实 AI 输出进行人工质量抽样，判断其是否适合演示、内部试用或进入进一步上线准备。

本 Runbook 不替代自动化测试。自动化测试验证结构、接口、异常和回归；本 Runbook 验证真实 AI 输出的业务质量、风格和合规边界。

---

## 2. 适用范围

适用：

- MVP 演示前抽样
- 内部试用前抽样
- Prompt 调整后抽样
- 真实 AI 服务切换后抽样

不适用：

- 自动化单元测试替代
- 法律合规最终审查
- 大规模 A/B 测试
- 真实微信发布审批

---

## 3. 前置条件

执行抽样前必须满足：

| 条件 | 要求 |
|---|---|
| 代码状态 | moments MVP 已合并到 `main` |
| 自动化回归 | moments 回归通过 |
| 服务状态 | Streamlit 和 FastAPI 可访问 |
| AI 模式 | 使用默认真实 AI，不设置 `MOMENTS_AI_MODE=mock` |
| 安全边界 | 不读取、不输出 `.env`、API Key、Token |
| 不做范围 | 不接真实微信、不自动发布、不定时发布 |

---

## 4. 抽样规模

最低抽样要求：

| 维度 | 要求 |
|---|---|
| 总样本数 | 至少 9 条 |
| 目标客户 | 跨境电商卖家、货物贸易、服务贸易各至少 3 条 |
| 文案风格 | 专业、轻松、销售感强各至少 3 条 |
| 产品卖点 | 到账快、费率透明、合规安全均需覆盖 |
| 异常样本 | 至少 1 条包含较长补充说明，验证稳定性 |

如用于正式对外演示，建议扩展到 15 条。

## 6. 质量评分标准

每条样本按 5 分制评分。

| 维度 | 5 分 | 3 分 | 1 分 |
|---|---|---|---|
| 结构完整 | 五类输出完整 | 缺 1 个辅助字段 | 缺正文或多个核心字段 |
| 私域自然度 | 像真实朋友圈表达 | 略像营销模板 | 明显硬广 / 群发 |
| 专业性 | 能体现跨境支付理解 | 有泛泛行业词 | 与行业无关 |
| 客户匹配 | 明确贴合目标客户 | 仅弱相关 | 目标客户错配 |
| 合规安全 | 无绝对化承诺 | 有轻微风险表达 | 出现保证收益、最低、最快等高风险 |
| 可复制性 | 可直接复制或轻微编辑 | 需要明显编辑 | 不适合发布 |

总分建议：

| 总分 | 判定 |
|---|---|
| 26-30 | 通过 |
| 21-25 | 有条件通过，可人工微调 |
| 20 以下 | 不通过，需要调整 Prompt / 解析 / 兜底策略 |

## 7. P0 / P1 问题判定

### P0：阻塞演示

出现任一情况即阻塞：

- 生成内容为空。
- 正文完全不可读。
- 输出明显与目标客户无关。
- 出现真实微信自动发布、定时发布等本期禁止能力描述。
- 输出 API Key、Token、密钥或敏感配置内容。

### P1：需修复后再演示

出现任一情况建议暂停演示：

- 出现“最低费率”“最快到账”“保证收益”“绝对安全”等绝对化承诺。
- 合规提示与正文矛盾。
- 多条输出高度重复，无法体现用户输入差异。
- 服务贸易、货物贸易、跨境电商卖家场景明显混淆。

### P2 / P3：可记录后续优化

- 文案略长。
- 语气略销售。
- CTA 不够自然。
- 标题不够有吸引力。

## 11. 通过门槛

允许演示：

- 样本数不少于 9 条。
- P0 = 0。
- P1 = 0。
- 平均分不低于 24。
- 每个目标客户至少 2 条通过或有条件通过。

允许内部试用：

- 样本数不少于 15 条。
- P0 = 0。
- P1 = 0。
- 平均分不低于 25。
- 兜底率、失败率和延迟指标已记录。

如不满足门槛：

- 切换 Mock 用于稳定演示。
- 将问题反馈给 AI 工程师调整 Prompt / 解析 / 兜底。
- 不进入无人值守真实用户试用。

---

## 12. 禁止事项

- 不输出 API Key、Token、`.env` 内容。
- 不将 AI 输出直接视为法律或金融建议。
- 不为了通过抽样而关闭合规提示。
- 不将真实微信发布、自动发布、定时发布纳入本抽样。
- 不把抽样通过等同于生产上线通过。

---

## 13. 下一步

建议执行顺序：

1. 产品负责人和 QA 按本 Runbook 抽样 9 条。
2. 如通过，允许演示继续使用真实 AI。
3. 如不通过，回退 Mock 演示并交 AI 工程师修复。
4. 进入内部试用前扩展到 15 条以上抽样。
5. 后续接 `AI-GOV-01` 指标记录和 `AI-GOV-02` smoke 脚本。
