# Ksher AgentAI 智能工作台

> **赛道**：赛道1 · 生产关系重构 & 效率提效
> **Slogan**：一人公司，AI武装 — 一个人 + AI，打赢一个团队的仗
> **定位**：不是原型演示，是团队和代理商每天在用的生产工具

---

## 一、问题的本质：渠道扩张的结构性瓶颈

### 问题不是"效率低"，是"规模化不可能"

Ksher正在建设「城市代理商」渠道体系。代理商以个人身份运营，帮Ksher拓展跨境支付商家。

渠道负责人面临的核心矛盾：

```
每新增1个代理商，你需要投入：
  ├── 培训周期：产品知识+话术+异议处理，至少2-3周才能独立拜访
  ├── 持续物料支持：方案/海报/PPT/内容，代理商无法自己产出
  ├── 产品答疑：40+文档的知识量，微信群实时回复
  └── 一对一陪跑：拜访前准备、拜访后复盘

你的时间是线性的，代理商是要规模化的：
  5个代理商 → 你能亲自带
  20个代理商 → 你带不过来
  100个代理商 → 物理上不可能

→ 结构性死结：渠道规模化的天花板 = 赋能团队的人力上限
```

### 双战场让问题更复杂

代理商需要在两个战场同时作战，赋能需求加倍：

| 战场 | 对手 | 获客逻辑 | 赋能要求 |
|------|------|---------|---------|
| **增量战场**（占90%市场） | 银行电汇 | 用数据撬动惰性："帮你算一笔没算过的账" | 精确成本对比+隐性成本分析 |
| **存量战场**（占10%市场） | PingPong/万里汇等 | 差异化切入：东南亚本地牌照+更低费率 | 竞品对比+定制方案 |

两个战场意味着**两套完全不同的话术、分析工具和物料**。

### 当前赋能瓶颈（真实现状）

| 赋能环节 | 现状 | 核心痛点 |
|---------|------|---------|
| 话术培训 | 渠道团队亲自带，每人2-3周 | 无法并行，新代理商排队等 |
| 客户方案 | 渠道团队协助制作 | 半天做一份，5个代理商同时要就排不过来 |
| 物料制作 | 依赖产品/设计排期 | 等1周出一张海报，代理商等不起 |
| 产品答疑 | 微信群实时回复 | 40+文档的知识量，不可能24小时在线 |
| 成本对比 | 手动计算 | 每个客户情况不同，一对一算太耗时 |

> **本质**：一个人（+有限团队）要完成市场+销售+培训+客服+数据+设计6个角色的工作，还要同时支持越来越多的代理商。这不是能力问题，是物理极限。

## 二、解决方案：改变渠道扩张的成本结构

### 核心洞察

大平台（PingPong/万里汇）的利润结构决定了它们只能服务大代理+大商家。中小代理+中小商家这个市场，它们不是不想做，是做不划算。

**AgentAI让代理赋能的边际成本趋零** — Ksher可以经济地服务无限多的代理商，包括大平台放弃的那些。这不是效率优化，是开辟了一个竞品成本结构无法覆盖的市场。

```
Before：
  渠道负责人
    ├── 亲自培训代理商A（投入2-3周）
    ├── 同时给A做方案、回答B的问题、帮C做海报…
    └── 天花板：你能同时支持的代理商 ≤ 5个

After：
  AgentAI 智能工作台
    ├── 代理商自助获取全套作战武器（话术+方案+物料+知识）
    ├── 同时支持任意数量的代理商
    └── 你的时间从"做执行"释放为"做决策"

成本结构改变：
  传统：赋能成本 ∝ 代理商数量（线性增长）
  AI后：赋能成本 ≈ 常数（API费用 ~¥2,000-5,000/月）
```

### 产品定位

**Ksher AgentAI 智能工作台** = 渠道赋能的基础设施。

不是7个独立AI小工具打包，而是一个**以"客户拜访"为核心的统一作战系统**——代理商和渠道团队每天打开就用的工作平台。

## 三、产品设计：以真实工作流为核心

### 三层产品结构

| 层级 | 功能 | 定位 | 生产就绪度 |
|------|------|------|-----------|
| **第一层：一键备战** | 输入客户 → AI实时生成话术+成本对比+方案+异议预判，支持多轮迭代优化 | 核心体验，日均使用频率最高 | 已上线，7个Agent全部实现，支持对话式迭代 |
| **第二层：独立工具** | 内容工厂 / 短视频中心 / 知识问答 / 海报设计 / 异议模拟，全部AI实时生成 | 日常运营，按需使用 | 已上线，15个Agent路由，全部真实生成 |
| **第二层+：内容精修** | 所有AI生成内容支持「去AI味」一键改写 + 自然语言多轮修改 + 版本管理 | 内容质量保障 | 已上线，7类AI痕迹诊断+对话式迭代 |
| **第三层：学习飞轮** | 反馈数据持久化 → 拜访结果追踪 → 成功案例自动注入 → 系统持续进化 | 长期壁垒，数据积累后价值递增 | 反馈持久化已实现，飞轮闭环持续迭代 |

### 代理商真实一天：AgentAI如何介入

这不是假设场景，是基于渠道负责人对代理商日常工作的直接观察：

| 时间 | 代理商在做什么 | 没有AgentAI | 有AgentAI |
|------|-------------|------------|----------|
| **08:30** 确认今日拜访名单 | 查看3个待拜访客户 | 联系渠道负责人要方案，等回复 | 直接打开AgentAI |
| **09:00** 准备第一个客户 | 深圳外贸工厂，用银行电汇 | 自己翻资料+微信问产品细节，准备2小时 | **一键备战**输入客户画像，90秒出全套作战包 |
| **10:00** 拜访中被问住 | "你们和PingPong有什么区别" | 含糊回答，回去问渠道负责人 | 拜访前已有异议预判+3种回复策略 |
| **11:30** 客户追问技术细节 | "泰国B2C到账要多久？" | 翻微信记录找答案 | **知识问答**秒级精准回答 |
| **14:00** 准备第二个客户 | 义乌电商卖家，用PingPong | 又要重新准备一套材料 | 一键备战**自动切换存量战场策略** |
| **16:00** 客户要正式提案 | "发一份方案给我看看" | 联系总部做PPT，等3天 | **PPT生成**，2分钟出专业方案 |
| **17:30** 发朋友圈维护 | 不知道发什么 | 转发公司公众号 | **内容工厂**生成本周内容，一键去AI味 |
| **18:00** 拍短视频 | 不会做/不知道发什么 | 放弃短视频渠道 | **短视频中心**全流程引导（选题→脚本→素材→发布） |
| **18:30** 标记拜访结果 | — | 微信群报告，信息零散 | 2步标记，数据自动积累进飞轮 |

> 代理商一天中至少6个环节被AgentAI覆盖。**"拜访准备"和"产品答疑"**是频率最高、痛感最强的两个——这正是一键备战和知识问答被设计为核心功能的原因。**短视频和内容营销**是增长最快的获客渠道——短视频中心让零经验的代理商也能照做就发。

### 渠道负责人视角：管理者的效率释放

| 你现在做的事 | 每周投入 | AgentAI后 |
|------------|---------|----------|
| 微信群回答代理商产品问题 | 7-14小时（碎片化） | 知识问答7×24自动回复，你只处理例外问题 |
| 帮代理商做客户方案 | 10-30小时（每份2-4h） | 代理商自助生成，你抽查把关 |
| 培训新代理商话术 | 每人占用2-3周 | AI提供话术+异议模拟，你只做关键节点陪跑 |
| 制作营销物料 | 2-3小时+协调设计 | 内容工厂+海报生成，代理商自助 |
| **释放出的时间用于** | — | **渠道策略、大客户攻关、代理商筛选、数据复盘** |

### 第一层：一键备战

代理商**一次输入**，AI编排4个引擎协同工作：

```
输入：客户公司 + 行业 + 目标国 + 月流水 + 当前收款方式 + 痛点

        ┌──────────────┐
        │  AI 编排中枢   │  ← 自动判断战场类型
        └──────┬───────┘
  ┌──────┬─────┼──────┬──────┐
  ▼      ▼     ▼      ▼      ▼
话术    成本   方案    异议   战场判断
引擎    引擎   引擎    引擎
  │      │     │      │
  └──────┴─────┴──────┘
              ▼
        完整「作战包」
```

**输出4件武器**：

| 武器 | 内容 | 解决什么问题 |
|------|------|------------|
| 定制话术 | 30秒破冰 + 3分钟讲解 + 微信跟进 | "不知道怎么开口" |
| 成本对比 | 5项总成本分析 + 可视化图表 + "年省¥XX" | "客户不信我算的账" |
| 定制方案 | 行业洞察→痛点→方案→产品推荐→接入流程 | "只会给报价，不会给方案" |
| 异议预判 | Top 3异议 + 每个3种回复策略 | "被问住了不知道怎么答" |

**双战场智能适配**（系统根据客户当前收款方式自动判断）：

| 判断条件 | 战场 | 话术策略 | 成本重点 |
|---------|------|---------|---------|
| 银行电汇 | 增量 | "帮你算一笔没算过的账" | 银行隐性成本（汇损+时间成本） |
| PingPong等 | 存量 | "东南亚这块，我们有硬差异" | 竞品对比（费率差+到账速度差） |
| 未选定 | 教育 | "跨境收款有3个坑" | 全景对比 |

### 第二层：独立工具（全部AI实时生成，非固定模板）

> **设计原则**：所有模块的输出均由AI Agent实时生成，而非从预设模板中填充。相同的输入在不同场景下产出不同内容——这是AI的核心价值。系统同时保留Mock降级能力，确保API异常时仍可演示。

**内容工厂**（市场专员 · AI获客内容中心，15个Agent协同）：

| 能力 | 输出 | 效率变化 |
|------|------|---------|
| 朋友圈7天计划 | 一周7条（文案+配图+发布时间），7种模式（日历/快速/素材转/改写/诊断/热点/流水线） | 数小时→5分钟 |
| 短视频中心 | 完整SOP工作流：选题→脚本→封面+字幕→制作指导→发布方案→竞品分析 | 从零开始→照做就发 |
| 一键内容生产 | 多Agent流水线（调研→写作→编辑→适配），自动产出多平台内容 | 4小时→10分钟 |
| 去AI味+多轮修改 | 所有生成内容一键去AI味（7类诊断改写）+自然语言迭代修改+版本管理 | 人工改写→一键+对话 |
| 海报/PPT | Pillow动态海报+AI生成PPT大纲+视频封面图 | 1-2小时→30秒 |

**短视频中心**（7子功能完整工作流，知识库驱动）：

代理商做短视频最大的障碍不是"不想做"，而是"不知道怎么做"。短视频中心不生成视频，而是提供从选题到发布的完整方法论+素材+工具引导：

| 子功能 | 输出 | 说明 |
|--------|------|------|
| 一键全流程 | 选题+脚本+发布方案一键串联 | 新手向导模式 |
| 选题策划 | 5-7个选题建议+周内容日历 | AI+知识库驱动 |
| 脚本创作 | 带【MM:SS-MM:SS】时间标记的完整脚本 | 抖音口语化 vs 视频号专业化 |
| 素材准备 | 封面图+SRT字幕+分镜脚本 | HTML→PNG，一键导出 |
| 制作指导 | 拍摄/剪辑/工具推荐（剪映/可灵AI等） | 纯知识库，无LLM调用 |
| 发布优化 | 标题+标签策略+发布时间+检查清单 | 分平台差异化 |
| 竞品分析 | 转录→结构分析→Ksher版改写+风格模仿 | 学习竞品优点 |

**知识问答**（多轮对话式交互，KnowledgeAgent驱动）：

基于40+文档的Ksher专属知识库，三层知识融合：
1. **Ksher产品知识**（最高优先）：费率/流程/牌照/竞品数据
2. **模型行业知识**（补充）：市场趋势/政策/通用商业知识
3. **融合输出**：既有精准产品数据，又有行业背景说服力

**多轮对话能力**：代理商可以像和专家聊天一样连续追问。例如先问"泰国B2B收款费率多少？"，再追问"那和PingPong比呢？"，AI理解上下文自动关联。支持行业和国家筛选，精准缩小知识检索范围。

输出的是"专家级分析"，不是"知识库摘抄"。每个回答附带置信度评级（高/中/低）、Ksher优势亮点提炼、话术建议、以及引用来源——代理商可直接用于客户沟通。

**海报/PPT生成**（DesignAgent AI生成）：Pillow动态生成品牌海报 + AI生成PPT大纲（根据客户行业/痛点/产品定制幻灯片结构和内容，非固定模板）

**异议模拟器**（ObjectionAgent驱动）：
- 查手册模式：AI实时生成3种回复策略（直接回应/共情引导/数据说服）
- AI对练模式：AI扮演客户实战训练，根据代理商的回答动态调整提问难度
- 自由练习模式：输入任意客户异议，AI即时生成定制化应对策略

### 第三层：学习飞轮（反馈持久化 → 数据闭环 → 持续进化）

```
使用 → 拜访 → 标记结果（签约/跟进/放弃+原因）→ 数据持久化存储
  │                                                    │
  │    ┌───────────────────────────────────────────────┘
  │    ▼
  │  反馈数据分析引擎
  │    ├── 话术效果排名（哪个版本转化率最高）
  │    ├── 行业转化率（电商 vs 外贸 vs 服务贸易）
  │    ├── 异议分布（最常遇到的Top 10）
  │    ├── 知识问答准确率（"有帮助/需改进"统计）
  │    └── 客户流失原因归因
  │    │
  │    ▼
  └── 成功案例自动注入 → 下一次生成更精准的作战包
```

**飞轮的三个关键机制**：

| 机制 | 实现方式 | 作用 |
|------|---------|------|
| **反馈数据持久化** | 所有模块的"有帮助/需改进"反馈 + 用户评论，持久化存储到 `data/feedback/` 目录，按模块分类（qa/battle_packs/content） | 积累训练数据，不再"用完即弃" |
| **拜访结果追踪** | 作战包使用后，代理商标记拜访结果（签约成功/需要跟进/客户拒绝）+ 原因 + 备注 | 建立"什么策略在什么场景有效"的因果数据 |
| **成功案例自动注入** | 标记为"签约成功"的作战包自动成为同行业/同战场的参考案例，下次生成时注入Prompt | 飞轮闭环：成功经验自动复用，系统越用越聪明 |

这不是静态工具，是一个**会随使用量增长而持续进化的系统**。每一次代理商的使用和反馈，都在让下一次的输出更精准。

## 四、技术架构

### 当前架构（已上线运行）

```
┌─────────────────────────────────────────────────────┐
│     Streamlit 前端（8个页面，Apple风格UI，多轮对话交互）  │
├─────────────────────────────────────────────────────┤
│             AI 编排中枢（BattleRouter + Pipeline）      │
│   战场判断 → 模型路由 → 半并行调度 → 结果组装+缓存      │
├──────┬──────┬──────┬──────┬──────────────────────┤
│ 话术  │ 成本  │ 方案  │ 异议  │ 内容/设计/知识问答      │
│Agent │Agent │Agent │Agent │ Agent（独立调用）        │
├──────┴──────┴──────┴──────┴──────────────────────┤
│   短视频中心（4 Video Agents）│ 内容流水线（4 Pipeline）│
│   选题/脚本/发布/分析         │ 调研→写作→编辑→适配    │
├─────────────────────────────────────────────────────┤
│   内容精修组件（RefinerAgent）                          │
│   去AI味（7类诊断改写）+ 多轮修改（对话式迭代）          │
├─────────────────────────────────────────────────────┤
│       多模型统一客户端（LLMClient，51个Agent路由）      │
│  单轮调用（call_sync）  +  多轮对话（call_with_history） │
│  Kimi K2.5（创意型）     Claude Sonnet 4.6（精准型）    │
├─────────────────────────────────────────────────────┤
│  知识库（39文档/46标签/7国）│ 费率引擎（纯Python计算）  │
├─────────────────────────────────────────────────────┤
│  结果缓存（5min TTL）  │  错误处理（3次重试+降级）     │
├─────────────────────────────────────────────────────┤
│  反馈持久化（FeedbackPersistence）│ Mock/Real自动切换  │
└─────────────────────────────────────────────────────┘
```

### 多模型协同：用对的模型做对的事

| 任务类型 | 模型 | 理由 | 月成本占比 |
|---------|------|------|----------|
| 话术/内容/设计/异议 | **Kimi K2.5** | 创意型中文生成，成本低 | ~30% |
| 成本计算/方案/知识问答 | **Claude Sonnet 4.6** | 精确推理，不编造数据 | ~70% |

混合方案月成本约¥800-2,000（20个代理商规模），比单一模型省60-70%。

### 生产可靠性保障

| 机制 | 实现 | 作用 |
|------|------|------|
| **结果缓存** | 5分钟TTL + 相似画像匹配 | 相同客户秒级响应，减少API调用 |
| **重试+降级** | 3次指数退避重试，Sonnet失败自动降级到Kimi | 单点故障不影响整体可用性 |
| **半并行执行** | 话术/成本/异议3个Agent并行，方案Agent串行等待成本数据 | 总响应时间<90秒 |
| **错误隔离** | 单Agent失败不阻塞整个作战包 | 3/4成功仍可用，失败模块显示重试按钮 |
| **知识库热更新** | Markdown文件编辑即生效，无需重启 | 费率调整/产品更新零停机 |
| **Mock/Real自动切换** | API正常→AI真实生成；API异常→自动降级到Mock模式并显示提示 | 任何情况下系统都可用，不会白屏 |
| **反馈数据持久化** | 用户反馈（有帮助/需改进/拜访结果）持久化到本地存储 | 数据不丢失，飞轮可持续积累 |

### 多轮对话架构

传统AI工具是"一问一答"——输入一次，输出一次，不满意就从头来。AgentAI采用**对话式交互**，用户可以在首次输出的基础上持续优化：

```
用户："泰国B2B收款费率多少？"
  → KnowledgeAgent 返回费率详解 + Ksher优势 + 话术建议

用户追问："那和PingPong比呢？"
  → AI 理解上下文，自动关联上一轮的泰国B2B场景，输出对比分析

用户再追问："如果客户月流水50万呢？"
  → AI 在前两轮基础上，给出具体费率阶梯和成本计算
```

**技术实现**：LLMClient新增`call_with_history()`方法，支持传入完整对话历史（`messages: [{role, content}]`），各Agent保持上下文连贯性。Session State存储每个模块的对话历史，页面刷新不丢失。

**覆盖范围**：
- 知识问答：多轮追问，AI理解上下文自动关联
- 作战包：生成后可对话式优化单个模块（"话术再强硬一些"/"成本对比加上时间成本"）
- **内容精修（全局）**：所有AI生成内容均嵌入「去AI味+多轮修改」组件——一键消除AI写作痕迹（7类诊断：套话/排比/过渡词/空洞形容词/机械节奏/空洞结尾/缺少人称），然后用自然语言指令迭代微调（"语气再活泼一些"/"加入泰国市场数据"），支持版本管理和历史回溯

### 持续迭代方向

| 维度 | 当前 | 下一步（基于使用反馈决定优先级） |
|------|------|--------------------------|
| 前端 | Streamlit | 如果移动端需求强烈 → Next.js重构 |
| 知识库 | Prompt注入（37文档） | 如果文档量超过100K tokens → 向量数据库RAG |
| 数据层 | 本地JSON | 如果多用户并发成为瓶颈 → PostgreSQL |
| 部署 | 本地/Streamlit Cloud | 稳定后 → 云服务器+CDN |

> 技术选型遵循"够用就好，问题驱动升级"原则。当前架构可以支撑20+代理商的日常使用，遇到真实瓶颈再升级，不做预防性过度工程。

## 六、竞争差异性

### 三层结构性优势

| 层级 | 创新点 | 可复制性 |
|------|--------|---------|
| **体验** | "一键备战"统一入口，一次输入全套输出 | UI可复制，但战场适配需要深度业务理解 |
| **成本结构** | 边际成本趋零，可服务大平台放弃的中小代理 | 大平台利润结构决定它们不会这样做 |
| **数据壁垒** | 飞轮积累"什么话术在什么场景转化率最高" | 没有Ksher代理商生态就没有这个数据 |

### vs 通用AI

| 维度 | ChatGPT/Claude | AgentAI |
|------|---------------|---------|
| 知识 | 不懂Ksher产品细节 | 39文档专属知识库，三层知识融合 |
| 输出 | 每次一件事 | 一次全套作战包（4件武器协同） |
| 双战场 | 不理解银行vs竞品 | 自动判断+策略切换 |
| 交互方式 | 单轮问答，每次重新描述需求 | 多轮对话+去AI味+迭代修改，AI记住上下文 |
| 内容质量 | 明显AI味 | 一键去AI味（7类诊断改写），多轮修改精调 |
| 短视频 | 只能生成脚本 | 完整SOP工作流（选题→脚本→素材→制作指导→发布优化） |
| 使用门槛 | 要会写Prompt | 填表即用 |
| 学习能力 | 每次对话独立，不记忆历史 | 反馈持久化+成功案例自动注入，越用越精准 |

### vs 竞品自建

大平台服务大代理，那是锦上添花。AgentAI服务所有代理商（含零经验新人），这是雪中送炭——没有AI就无法运作的一人公司模式。

## 七、落地路径

### 这不是"做完Demo再想落地"——系统已在运行

当前状态：**15个Agent已实现，8个页面全部可操作，知识库39文档已接入**。这不是原型，是一个可以直接给代理商用的工具。

### 推进计划

```
当前 · 系统已可用（4月）
  └── 全功能已实现，渠道负责人自己先用
      状态：7个Agent / 6页面 / 37文档知识库 / 错误处理+缓存

5月 · 黑客松展示 + 种子用户启动
  └── 黑客松展示系统能力
  └── 同步选择3-5个种子代理商开始实际使用
      选人标准：
      ├── 2个新手（入行<3月）→ 验证"零经验也能用"
      ├── 2个老手（入行>6月）→ 验证"比现有方式好"
      └── 1个高频拜访（周>5次）→ 验证重度场景

6月 · 反馈驱动迭代
  └── 基于真实使用数据迭代
      跟踪指标：
      ├── 周活跃率（每周至少用1次的代理商比例）
      ├── 功能使用分布（哪个功能用得最多/最少）
      ├── 知识问答准确率（代理商反馈"有用/没用"）
      └── 拜访准备时间变化（代理商自报告）

7-8月 · 全量推广
  └── 全体代理商接入
      目标：月均调用 ≥ 200次，代理商周活 ≥ 60%

Q4 · 飞轮启动
  └── 数据积累到可分析的量级
      目标：基于数据推荐 vs 通用推荐，转化率差异 ≥ 10%
```

### 反馈收集机制

| 方法 | 频率 | 收集什么 |
|------|------|---------|
| 系统埋点 | 实时 | 功能使用频率、页面停留时间、跳出率 |
| 代理商电话回访 | 每周15分钟 | 哪些功能用了？没用的为什么？遇到什么问题？ |

| 满意度问卷 | 每月 | NPS评分+开放式反馈 |

| 层级 | 标准 | 含义 |
|------|------|------|
| 最低线 | 代理商愿意继续用 | 不是"试一次就放弃" |

"我就是渠道负责人" = 落地路径最短。不需要跨部门协调，不需要产品经理排期。我自己就是用户、推动者和验证者——系统好不好用，我每天都在检验。
