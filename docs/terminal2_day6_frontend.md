## 你的任务

### P0：UI 最终检查与修复

1. **全页面视觉一致性检查**
   启动应用，逐个检查 6 个页面：
   ```bash
   streamlit run app.py
   ```
   - 一键备战：表单/按钮/Tab/作战包展示
   - 内容工厂：生成结果卡片/复制按钮
   - 知识问答：预设问题/回答/反馈按钮
   - 异议模拟：训练模式/结果展示
   - 设计工作室：海报预览/PPT大纲
   - 仪表盘：Plotly 图表/KPI卡片

2. **修复发现的视觉问题**
   检查以下常见问题：
   - [ ] 是否有白色文字在白色背景上不可见？
   - [ ] 是否有深色背景元素突兀？
   - [ ] 按钮是否为药丸形状？
   - [ ] 卡片是否有不协调的阴影？
   - [ ] 输入框聚焦状态是否正常？
   - [ ] 侧边栏文字是否清晰可读？

3. **响应式布局验证**
   - 浏览器开发者工具 → iPhone 375px 宽度
   - 检查：侧边栏可收起、表单单列、Tab 可横向滚动

### P1：演示体验优化

4. **一键备战页面优化**
   - 表单区域：确保字段标签清晰、间距舒适
   - 生成按钮：突出显示，使用 `type="primary"`
   - 加载状态：spinner 文字简洁，避免技术术语
   - 作战包展示：4 个 Tab 切换流畅，内容排版整齐

5. **仪表盘页面优化**
   - KPI 卡片：数值突出，标签清晰
   - Plotly 图表：确认浅色主题下文字可读
   - 检查图表标题、坐标轴标签颜色

6. **页面加载速度检查**
   - 首次加载时间（目标 < 3 秒）
   - 页面切换是否流畅
   - 如有大图（logo），确认已优化

### P2：演示材料准备

7. **截图准备**
   - 每个页面截取 1-2 张高质量截图
   - 重点：一键备战（表单+结果）、仪表盘
   - 保存到 `docs/screenshots/`

8. **演示脚本配合**
   - 阅读 `docs/demo_script.md`
   - 确认 UI 流程与脚本一致
   - 如有 UI 流程问题，提出修改建议
