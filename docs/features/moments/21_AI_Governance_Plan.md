---

## 2. 治理目标

| 目标 | 说明 |
|---|---|
| 质量可控 | 明确真实 AI 输出质量抽样标准和不通过处理方式 |
| 成本可见 | 记录调用次数、失败率、兜底率和单次调用成本估算 |
| 限流可控 | 明确用户级、session 级、环境级限流策略 |
| 故障可兜底 | 超时、空响应、格式异常、敏感内容均能走安全兜底 |
| 模式可切换 | Mock / 真实模式切换规则明确 |
| 日志可复盘 | 保留最小可复盘日志，不记录敏感信息 |
| 密钥安全 | 不读取、不输出、不提交 API Key、Token、`.env` |

---

## 3. AI 模式策略

| 模式 | 触发方式 | 用途 | 是否适合 QA |
|---|---|---|---|
| 真实 AI | 默认模式，未设置 `MOMENTS_AI_MODE=mock` 且无 `mock:*` 标记 | 演示、真实业务试用 | 适合端到端 smoke，不适合固定断言 |
| 全局 Mock | `MOMENTS_AI_MODE=mock` | 自动化回归、稳定演示、无外部依赖验证 | 适合 |
| 场景 Mock | `extra_context` 包含 `mock:success` / `mock:error` / `mock:empty` / `mock:sensitive` | 复现异常、兜底、敏感内容等固定场景 | 适合 |

原则：

- 自动化回归优先使用 Mock。
- 真实 AI 仅用于 smoke、抽样和演示。
- 不允许为 QA 关闭限频或绕过安全边界。

---

## 4. 质量验收标准

真实 AI 输出每次抽样至少检查以下字段：

| 检查项 | 合格标准 |
|---|---|
| 结构完整 | 包含标题、正文、转发建议、合规提示、改写建议 |
| 字数 | 朋友圈正文建议不超过 300 字 |
| 私域表达 | 语气自然，不像硬广或群发模板 |
| 专业性 | 能体现跨境收款、合规、安全、资金效率等行业理解 |
| 目标客户匹配 | 能区分跨境电商卖家、货物贸易、服务贸易 |
| 合规边界 | 不出现最低、最快、保证收益、绝对安全等绝对化承诺 |
| 兜底行为 | 失败时明确提示需要人工补充，不伪装成最终可发布内容 |

建议抽样规则：

- 每次演示前抽样 5 条。
- 每个目标客户至少 1 条。
- 每种文案风格至少 1 条。
- 如任一 P0/P1 质量问题出现，暂停真实 AI 演示，切回 Mock 或修复 Prompt / 解析策略。

## 5. 成本与调用监控

MVP 后续建议记录以下指标：

| 指标 | 用途 |
|---|---|
| `request_count` | 总调用量 |
| `success_count` | 成功生成量 |
| `fallback_count` | 兜底触发量 |
| `error_count` | 失败量 |
| `timeout_count` | 超时量 |
| `avg_latency_ms` | 平均延迟 |
| `p95_latency_ms` | 长尾延迟 |
| `estimated_cost` | 估算成本 |

当前阶段不要求立即实现成本计费，只要求后续实现时不得记录：

- API Key
- Token
- `.env` 内容
- 完整用户敏感输入
- 真实客户隐私信息

---

## 6. 限流策略

当前 MVP 限频：

- 同一 `session_id` 10 秒内最多 1 次生成请求。

后续建议：

| 层级 | 建议策略 |
|---|---|
| Session 级 | 维持 1 次 / 10 秒，防止误点和重复请求 |
| IP / 设备级 | 演示环境可设置保守限流，如 30 次 / 小时 |
| 全局级 | 防止真实 AI 成本失控，如 300 次 / 日 |
| Mock 场景 | 不建议绕过限流；需要独立 session 验证异常场景 |

限流提示必须区分：

- 真实限频：提示稍后再试
- AI 失败：提示生成失败或使用兜底
- 格式异常：提示生成结果不完整并提供兜底

---

## 7. 故障兜底策略

| 场景 | 处理方式 | 前端表现 |
|---|---|---|
| AI 超时 | 返回 fallback | 显示需要人工补充 |
| AI 空响应 | 返回 fallback / `output_incomplete` | 显示生成结果不完整 |
| AI 非 JSON | 尝试 repair；失败则 fallback | 显示安全兜底 |
| 敏感内容 | 返回风险提示和改写建议 | 不建议直接发布 |
| HTTP 429 | 返回限频提示 | 提示稍后再试 |
| 网络失败 | 前端显示网络错误，可保留旧结果 | 不覆盖旧结果 |

兜底内容不得被包装成“已审核可发布”。

---

## 8. 日志与脱敏

日志最小字段：

- `request_id`
- `session_id` 的脱敏哈希或短标识
- `target_customer`
- `copy_style`
- `product_points_count`
- `status`
- `fallback_used`
- `error_code`
- `latency_ms`
- `created_at`

禁止记录：

- API Key
- Token
- `.env` 内容
- 完整 Prompt 原文
- 完整用户隐私输入
- 完整 AI 原始响应，除非经过脱敏并有明确调试开关

- 不输出 API Key、Token、SSH Key。
- 不自动部署生产。
- 不新增真实微信接口。
- 不新增自动发布朋友圈。
- 不扩大到 CRM、素材库、海报、数据分析闭环。
- 不为通过测试而关闭限频、安全检查或兜底逻辑。

真实 AI 当前已经可用，但生产化还需要治理。

建议下一步：

1. 先完成 `AI-GOV-03` 质量抽样 Runbook。
2. 再完成 `AI-GOV-01` 指标记录。
3. 再补 `AI-GOV-02` smoke 脚本。
