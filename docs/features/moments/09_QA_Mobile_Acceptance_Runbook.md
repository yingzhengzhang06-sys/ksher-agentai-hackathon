# 发朋友圈数字员工 - 移动端人工验收 Runbook

## 1. 阶段目标

- 完成 M-01 ~ M-10 移动端人工验收
- 验证功能实现、AI 输出、UI/UX、异常处理符合 PRD/MVP
- 记录截图、缺陷编号、复现步骤和责任角色
- 为最终 QA 验收结论提供可追溯证据

---

## 2. 角色职责

| 角色 | 职责 |
|------|------|
| QA / 测试负责人 | 执行 M-01~M-10 移动端人工验收，记录截图、缺陷编号 MOMENTS-QA-XX、复现步骤，更新文档 |
| 工程师 | 待命，仅按缺陷编号修复对应缺陷，并执行回归测试 |
| 产品负责人 | 监督验收结果，确认是否进入上线前审批 |
| 架构师 / 技术负责人 | 对技术性缺陷提供分析和支持，确认修复方案 |

---

## 3. 测试环境准备清单

- 移动端设备型号与系统版本列表
- 测试账号 / 用户信息
- 网络状态：Wi-Fi / 移动数据
- App 或 WebView 安装版本
- 截图工具准备
- 缺陷记录模板准备（MOMENTS-QA-XX）
- 测试用例文档：`07_Test_Cases.md`

## 7. 验收结论模板

- **通过**：核心链路可用，P0/P1 缺陷均已解决
- **有条件通过**：存在 P2/P3 缺陷，已记录，可后续修复
- **不通过**：存在 P0/P1 阻塞问题或实现了不允许的功能

> QA 完成后，将结论和缺陷列表提交产品负责人审批。

---

## 8. 文档更新要求

- 更新 `07_Test_Cases.md` 第 14 节：执行状态、截图、缺陷信息
- 更新 `08_Collaboration_Status.md`：当前阶段状态、缺陷总览、已完成/待修复任务
- 所有截图和缺陷编号必须在文档中可追溯

---

## 9. 禁止事项

- 不允许修改业务代码（前端 / 后端 / AI）
- 不允许增加本期 MVP 范围外的功能
- 不允许接入真实微信发布接口
- 不允许自动发布或定时发布功能
- 不允许直接上线或部署生产环境
- 不允许删除或覆盖现有测试用例或历史缺陷记录
